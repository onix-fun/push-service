# Архитектура Push Service

## Компоненты
1. **HTTP API:** Позволяет клиентам (мобильным приложениям, браузерам) регистрировать свои токены.
2. **Worker:** Слушает RabbitMQ на предмет команд на отправку уведомлений конкретным пользователям.
3. **Storage (PostgreSQL):** Хранит маппинг `recipient_id -> [device_tokens]`.
4. **Providers Gateway:** Абстракция над SDK провайдеров (Firebase SDK, APNS HTTP/2 и т.д.).

## Схема доставки
1. Внешний сервис публикует событие в `push.commands` RabbitMQ.
2. Worker получает команду, запрашивает в БД все активные токены для указанного получателя.
3. Worker группирует токены по провайдерам и вызывает соответствующие шлюзы.
4. Результат отправки публикуется в `push.delivery.events`.
